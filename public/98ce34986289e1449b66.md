# これなに

以前、[Laravel で Microsoft Teams に簡易的なメッセージを通知する手順](https://qiita.com/su3-hokkaido/items/65eb2aba34d2ea99d738)という記事を書きました。
先の記事では Office 365 の Incoming Webhook コネクタを使って Teams に通知を飛ばす方法を紹介していたのですが、[Microsoft がこのコネクタを廃止するという発表があったようです。](https://devblogs.microsoft.com/microsoft365dev/retirement-of-office-365-connectors-within-microsoft-teams/)（知らなかった。。。）

この記事では、廃止の背景と、後継であるPower Automate WorkflowsのWebhookを使ってLaravelからTeamsに通知を送る方法についてまとめます。


# 本記事を作成する際の情報の確認日

# この記事で出てくる用語説明

- [Teams](https://www.microsoft.com/en-us/microsoft-teams/group-chat-software): Microsoft社が提供しているチャットツール
- [Laravel](https://laravel.com/): PHPの有名な開発フレームワークの1つ
- [Power Automate](https://make.powerautomate.com/): Microsoft社が提供しているワークフロー自動化サービス
- Incoming Webhook: 外部アプリケーションからTeamsのチャンネルにメッセージを送信するためのコネクタ（旧方式、廃止予定）
- Adaptive Card: Microsoft が定義するカードUIのフォーマット、Power Automate Workflows の Webhook ではこの形式でメッセージを送信する必要がある


# Office 365 Incoming Webhookコネクタの廃止について

[Microsoftは2026年4月30日をもってOffice 365コネクタ（O365 Connectors）を廃止すると発表しています。](https://devblogs.microsoft.com/microsoft365dev/retirement-of-office-365-connectors-within-microsoft-teams/)
これにより、旧来のIncoming Webhookコネクタを使った通知は動作しなくなります。後継として推奨されているのが**Power Automate Workflows**のWebhookです。

![image.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/2819748/a66efbf2-fdf5-46a1-b425-143067260264.png)

### 旧方式と新方式の違い

| | 旧方式（Incoming Webhook） | 新方式（Power Automate Workflows） |
|---|---|---|
| コネクタ | Office 365 Incoming Webhook | Power Automate Workflows |
| ペイロード形式 | MessageCard | Adaptive Card |
| ステータス | 2026年4月30日を最後に廃止予定 | 推奨 |
| セットアップ | チャンネル設定から追加 | Power Automateでフロー作成 |

一番大きな違いは**ペイロードの形式**です。旧方式では`MessageCard`形式のJSONをPOSTするだけでよかったのですが、新方式では`Adaptive Card`形式のペイロードを送信する必要があります。

## 2. Laravelでカスタムログチャンネルを実装する

旧方式で`cbytedigital/laravel-teams-logger`のようなパッケージを使っていた場合、これらはMessageCard形式で送信するためPower Automate Workflowsでは動作しません。

Adaptive Card形式に対応したカスタムログハンドラを自前で実装します。

### `app/Logging/TeamsWebhookHandler.php`

```php
<?php

namespace App\Logging;

use Monolog\Handler\AbstractProcessingHandler;
use Monolog\Level;
use Monolog\LogRecord;

class TeamsWebhookHandler extends AbstractProcessingHandler
{
    private string $webhookUrl;

    private string $projectName;

    public function __construct(
        string $webhookUrl,
        string $projectName = 'Laravel',
        int|string|Level $level = Level::Debug,
        bool $bubble = true
    ) {
        parent::__construct($level, $bubble);

        $this->webhookUrl = $webhookUrl;
        $this->projectName = $projectName;
    }

    /**
     * Build Adaptive Card payload for Power Automate Workflows.
     */
    private function buildPayload(LogRecord $record): array
    {
        $level = $record->level->name;
        $message = $record->message;

        $facts = [
            ['title' => 'Level', 'value' => $level],
            ['title' => 'Message', 'value' => mb_substr($message, 0, 500)],
            ['title' => 'Time', 'value' => $record->datetime->format('Y-m-d H:i:s')],
        ];

        // 例外情報がある場合は追加
        if (
            isset($record->context['exception'])
            && $record->context['exception'] instanceof \Throwable
        ) {
            $exception = $record->context['exception'];
            $facts[] = ['title' => 'Exception', 'value' => get_class($exception)];
            $facts[] = ['title' => 'File', 'value' => $exception->getFile() . ':' . $exception->getLine()];
            $facts[] = [
                'title' => 'Trace',
                'value' => mb_substr($exception->getTraceAsString(), 0, 1000),
            ];
        }

        return [
            'type' => 'message',
            'attachments' => [
                [
                    'contentType' => 'application/vnd.microsoft.card.adaptive',
                    'contentUrl' => null,
                    'content' => [
                        '$schema' => 'http://adaptivecards.io/schemas/adaptive-card.json',
                        'type' => 'AdaptiveCard',
                        'version' => '1.4',
                        'body' => [
                            [
                                'type' => 'TextBlock',
                                'text' => "{$this->projectName} - Error {$level}",
                                'weight' => 'Bolder',
                                'size' => 'Medium',
                                'color' => 'Attention',
                            ],
                            [
                                'type' => 'FactSet',
                                'facts' => $facts,
                            ],
                        ],
                    ],
                ],
            ],
        ];
    }

    protected function write(LogRecord $record): void
    {
        $payload = $this->buildPayload($record);
        $json = json_encode($payload, JSON_UNESCAPED_UNICODE);

        $ch = curl_init($this->webhookUrl);
        curl_setopt($ch, CURLOPT_CUSTOMREQUEST, 'POST');
        curl_setopt($ch, CURLOPT_POSTFIELDS, $json);
        curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
        curl_setopt($ch, CURLOPT_TIMEOUT, 10);
        curl_setopt($ch, CURLOPT_CONNECTTIMEOUT, 3);
        curl_setopt($ch, CURLOPT_HTTPHEADER, [
            'Content-Type: application/json',
            'Content-Length: ' . strlen($json),
        ]);

        curl_exec($ch);
        curl_close($ch);
    }
}
```

### `app/Logging/TeamsLoggingChannel.php`

```php
<?php

namespace App\Logging;

use Monolog\Level;
use Monolog\Logger;

class TeamsLoggingChannel
{
    /**
     * Create a custom Monolog instance for Teams Power Automate Workflows.
     */
    public function __invoke(array $config): Logger
    {
        $handler = new TeamsWebhookHandler(
            webhookUrl: $config['url'] ?? '',
            projectName: $config['name'] ?? 'Laravel',
            level: $config['level'] ?? Level::Error,
        );

        return new Logger('teams', [$handler]);
    }
}
```

## 3. `config/logging.php`を設定する

旧方式のMessageCard形式のTeamsチャンネルを、新しいカスタムチャンネルに置き換えます。

```php
// config/logging.php

'channels' => [
    'stack' => [
        'driver' => 'stack',
        'channels' => ['daily', 'teams'],
        'ignore_exceptions' => false,
    ],

    // ... 他のチャンネル設定 ...

    'teams' => [
        'driver' => 'custom',
        'via' => \App\Logging\TeamsLoggingChannel::class,
        'level' => 'error',
        'url' => env('LOG_TEAMS_WEBHOOK_URL'),
        'name' => 'your-project-name',
    ],
],
```

## 4. `.env`にWebhook URLを設定する

Power Automateで発行されたURLを`.env`に設定します。

```.env
LOG_TEAMS_WEBHOOK_URL=https://prod-xx.westus.logic.azure.com:443/workflows/...
```

# 旧方式からの移行時に注意すること

### `cbytedigital/laravel-teams-logger`などのパッケージについて

旧方式の MessageCard 形式を前提としたパッケージは Power Automate Workflows では動作しませんでした。今回のように自前でカスタムハンドラを実装するか、 Adaptive Card 対応のパッケージに切り替える必要があるようです。

### ペイロード形式の違い

旧方式と新方式ではPOSTするJSONの構造が変わっていました。

```json
// 旧方式（MessageCard）
{
  "summary": "Error",
  "title": "Error",
  "sections": [{ "activityTitle": "Info", "facts": [...] }],
  "themeColor": "721C24"
}
```

```json
// 新方式（Adaptive Card）
{
  "type": "message",
  "attachments": [{
    "contentType": "application/vnd.microsoft.card.adaptive",
    "content": {
      "type": "AdaptiveCard",
      "version": "1.4",
      "body": [{ "type": "TextBlock", "text": "Error" }]
    }
  }]
}
```

### Adaptive Cardのペイロード上限

Power Automate Workflowsでは、ペイロードの最大サイズは**256KB**です。スタックトレースを含む場合は`mb_substr`などで切り詰めるようにしましょう。


# まとめ

| 項目 | 対応内容 |
|---|---|
| 廃止対象 | Office 365 Incoming Webhook コネクタ（2026年4月30日まで利用可、その後は廃止） |
| 後継 | Power Automate Workflows の Webhook |
| ペイロード形式 | MessageCard -> Adaptive Card |
| Laravelでの対応 | カスタムログハンドラを実装し`config/logging.php`を差し替え |
| 旧パッケージ | MessageCard 前提のため動作しない、自前実装またはAdaptive Card対応パッケージに切り替え |

上記のことからもわかる通り、新規で実装する場合は最初から Power Automate Workflows を使うのが無難です。既存プロジェクトでも、今回紹介したカスタムハンドラの実装量は少ないので、早めに移行しておくのが良いかなと思います。


# 最後に

前回の記事では「簡単にTeams通知できた！」と書いていたのですが、まさかコネクタ自体が廃止になるとは思っていませんでした。
今回の移行で一番ハマったポイントはペイロード形式の違いで、旧方式の`{ "text": "message" }`のような単純な JSON では Power Automate Workflows が受け付けてくれません。 Adaptive Card 形式が必須という点は移行時に見落としやすいので、同じ状況の方の参考になれば幸いです。

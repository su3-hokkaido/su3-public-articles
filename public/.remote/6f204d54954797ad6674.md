# なにこれ

Docker のキャッシュが多すぎてローカルマシンの容量を圧迫していたので削除したいと思ったので、そのメモを記事にしました。
なお、これは個人メモなので自分にとって必要十分な情報のみ記載しております。

# Docker とは

Docker とはアプリケーションを 「コンテナ」 という箱に詰め込んで、どこでも同じように動かせるようにするプラットフォームです。
本記事では Docker 自体については記載する予定はないので、必要があれば[公式サイト](https://www.docker.com/)をご覧ください。

# ディスク使用量の確認コマンド

現在のディスク使用量を確認できます。

```bash
docker system df
```

# キャッシュ削除コマンド一覧
## タグが付いていない、不要になった古いキャッシュのみを削除

下記のコマンドを実行すると Docker のイメージビルド時に生成される ビルドキャッシュを削除してディスク容量を確保します。

```bash
docker builder prune
```

## すべての未使用キャッシュを削除

`-a` を付与すると、現在使われていないビルドキャッシュをすべて削除します。

```bash
docker builder prune -a
```

## 確認なしで実行

`-f` を付けると、「本当に削除しますか？」という確認メッセージをスキップできます。

```bash
docker builder prune -f
```

# 注意点

- 削除されるもの: Dockerfile のビルド中に作成された中間データ（BuildKit キャッシュ）です。
- 削除されないもの: すでに完成している（タグが付いている）Dockerイメージや、現在実行中のコンテナなどは削除されません。

# これなに

- ローカルで複数アプリを開発していると「あれ？このポート使えないんだっけ？」みたいなことに出会います
- 特に全く別サービスを複数抱えていたりするとローカルで競合してしまうことがしばしば
- 毎回調べるのが面倒なので個人用にこのメモ記事を作成しました
- もし参考にされる方がいらっしゃればメモとして活用ください

# 前提条件

macOS または Linux で実行を想定しています。

# ポートとは

ネットワークの接続を開始したり終了したりする、OS上の仮想的な場所です。
簡単にいうとコンピュータとネットワークの間にあるドアみたいなものですね。（こういう話は[わわわIT用語辞典 @wa3_pcs](https://x.com/wa3_pcs)さんの記事を参考にするのがわかりやすい）

https://wa3.i-3-i.info/word1774.html

# ポートステータス確認

macOS の場合は lsof、Linux の場合は lsof に加えて ss および netstat が使えます。

### lsof の場合

コマンドは以下の通り

```bash
lsof -i :5173
```

サンプル出力結果は以下の通り。

```bash
% lsof -i :5173
COMMAND    PID         USER   FD   TYPE             DEVICE SIZE/OFF NODE NAME
com.docke 4680 su3-hokkaido  321u  IPv6 0xb74a90330ed8c1fb      0t0  TCP *:5173 (LISTEN)
%
```

### netstat, ss の場合 (通常 macOS では使えません)

昔は netstat, 最近は ss で確認するらしいです。
コマンドは以下の通りで、どちらかを使えばOKです。

```bash
ss -tlnp | grep 5173
netstat -tlnp | grep 5173
```

サンプル出力結果は以下の通り。

```bash
# ss
bash-5.2# ss -tlnp | grep 5173
LISTEN 0      511                *:5173             *:*    users:(("node",pid=63,fd=18)) 
bash-5.2#

# netstat
bash-5.2# netstat -tlnp | grep 5173
tcp6       0      0 :::5173                 :::*                    LISTEN      63/node             
bash-5.2# 
```

もしコマンドが見つからない場合はネットワークコマンドをインストールしてください。
例として yum でインストールするコマンドを下記の通り示します。

```bash
# ss コマンド
yum install -y iproute
                                    
# netstat コマンド
yum install -y net-tools
```

# ポート解放手順

前項のところで確認したPIDを `<PID>` に指定してプロセスを終了させて解放できます。

```bash
kill <PID>
```

解放したらステータスを確認しておきましょう、空っぽで返ってきたら無事に解放できています。（前項でいくつかコマンドを挙げたので、ここでは汎用的に使える lsof コマンドをサンプルとして再掲）

```bash
% lsof -i :5173
```
